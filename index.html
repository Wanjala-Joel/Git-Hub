Git Hub

GitHub is a global company that provides hosting for software development version control using Git.

Several commands used in Git Hub on a linux operating system

The commands have several stages in which one must use to host a website on Git Hub for version control. The following are some basic commands used in the terminal for hosting a repository. Each command must start with git in the terminal. Basic commands include:

  • git version

    • Helps to know which version of git is installed on a computer.

  • cd

    • This is a command where you copy paste the path containing the folder which you want to host in the github repository. Example cd /home/joel/documents/java

  • ls

    • This command is used to check the files that are resent in the folder you have chosen from your machine.

  • ll -a

    • Shows all that is contained in the folder.

  • git clone link

    • This clones the your files to the repository. The link is acquired from the repository you have created in the Git Hub.

  • git status

    • This command is used to show the untracked files in the repository ready to be pushed in the master or branch you are working from.

  • cd website/

    • This command is used to navigate the git repository you have created. Example cd Basic-of-JS/. Notice that a slash "/" punctuation is necessary at the end of the website.

  • git add .

    • This is used to add the files to the branch you are working in or the master

  • git status

    • This is used to show the files in the master or branch to be commited in your repository

  • git commit -m 'message'

    • This is used to commit the changes to the master or branch. The 'message' is where you type a message about the files you are willing to host.

  • git push origin master

    • This command pushes your files into the repository you have chosen.
